Summary

Within the PerfStat application there are two defined formats for sending data from the client to the server: info and data. Both adhere to a strict methodology for organizing the data. All general information related to the host are identified with the info data format. All information related to the performance or operation of a host are identified within the data format.

Info Format

The info format is used to send general data about a host. Things like the number of CPUs installed in the host are captured under this data format. Currently this is not a dynamic feature within PerfStat and users cannot define their own info categories. Below is a list of things that are currently gatherd:

CPU Model - The processor(s) model installed in the host

CPU Speed - The speed of the processor(s) in MHZ

Number of CPUs - The number of processor(s) installed in the host

Total Memory - Amount of memory installed in the host in MB

OS - The operating system type installed on the host

OS Version - The version of the operating system installed on the host

Kernel Version - The version of the operating system kernel installed on the system

Patch List - A comma separated list of patches installed on the host

The format used to send info from client to server is as follows:

[osType] info [cpuNum] [cpuModel] [cpuSpeed] [memTotal] [osVersion] [kernelVersion] [patches]

Data Format

The data format is used to send specific performance or operational information about a host. Things like cpu utilization are captured under the data format. This is a dynamic feature with PerfStat and users can define their own unique data categories. A data category is called a service. A service can be standalone like cpu or can have additional sub-services like fs. This was done to provide flexibility and granularity. Below is a list of services that are configured by default and are part of the PerfStat distribution:

CPU - Data about the utilization of a hosts CPU(s)

FS - Data about the utilization of a hosts file system(s)

IO - Data about the utilization of a hosts physical drive(s)

MEMORY - Data about the utilization of a hosts physical/virtual memory

PROCS - Data about the number of processes running on a host

SOCKET - Data about the utilization of a hosts socket connection(s)

TCP - Data about the utilization of a hosts network interface(s)

UPTIME - Data about the amount of time a host has remained up

The format used to send data from client to server is as follows:

[osType] data [service] [Metric Value1] [Metric Value2] [Metric Value3]...

osType - The os type can be the following: Linux, SunOS, or WwindowsNT

service - If it is a standalone service it would be something like "apache". If it is a service with sub-services then it would look something like "oracle.db1".

Metric Value# - This is the value of a metric being measured. If the service is apache and Metric1 is number of httpd processes the Metric Value might be "20". The metric names are associated to the metric values on the PerfStat server. You define the sequence on the server and then ensure the client program for that service sends the metric data in the correct order.

In order to add a unique user defined service into the PerfStat application you need to do the following:

1. Write client program for unique service that gathers data

2. Organize data using above format and interface with PerfStat client transport program perf.pl

3. Configure serialized object for unique service on the PerfStat server which details all the information about that service including the ordering of metrics
