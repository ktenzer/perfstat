Summary

The PerfStat user interface is responsible for displaying client data based on user configurations. The user interface is a collection of cgi programs located under the perfhome/cgi directory. Most of these programs are not compiled therefore ActiveStat perl 5.8 must be installed on the server. The cgi programs are interpreted via a web browser. The PerfStat application supports the Apache Web Server.

CGI Log

The PerfStat user interface has a separate log located under /var/logs/cgi.log. By analyzing the cgi.log you should be able to tell if the issue is related to the user interface. The cgi log uses the following format:

[Timestamp] [Client Program] [Alert Level] [Message]

[Fri Feb 4 16:52:16 2005] index.pl: ERROR: Couldn't open file handle to /perfstat/dev/1.52/server/var/events/cashcow.perfstat.com/conn.ping.log: No such file or directory

In the above example we see that on Feb 4 the index.pl cgi program encountered an error event, Couldn't open file handle to /perfstat/dev/1.52/server/var/events/testhost.test.com/conn.ping.log: No such file or directory

Alert Levels - The PerfStat cgi log has the following alert levels: DEBUG, INFO, WARNING, and ERROR.

DEBUG - By default debug log messages are not enabled. If debug is enabled much more information will be displayed in the cgi.log. You can enable debug by setting the DEBUG field to "1" in the perf-conf configuration file and then restarting the PerfStat server application.
