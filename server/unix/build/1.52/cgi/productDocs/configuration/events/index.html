Summary

Events are the key to status monitoring. Any change in status is considered an event. The four status types in PerfStat are: OK, CRIT, WARN, and nostatus. If event logging is enabled events will be written to a log. Each service has an event log containing events for all that services metrics. Event logging must be enabled in order to do alerting.

Enable Events

1) Ensure EVENTLOG is enabled in perf-conf and set to "Y" by running the following command:

/perfhome/bin/tools/perfconfig -list

2) To change any of the perf-conf settings run the following command:

/perfhome/bin/tools/perfconfig -server

Note: if you are logged in as root you will need to run the perfconfig command with "-root" flag. You can also simply cat the /perfhome/etc/perf-conf file.

Configure Event Log Size

The size is the number of lines in the event log file. The event log will grow to the size (lines) of the LOGSIZE value in perf-conf and then will truncate old log entries to make room for new log entries when it reaches that size.

1) Set LOGSIZE to the maximum size (lines) that you want the event logs to be by running the following command:

/perfhome/bin/tools/perfconfig -server

Note: if you are logged in as root you will need to run the perfconfig command with "-root" flag. You can also simply vi the /perfhome/etc/perf-conf file.

Configure Event Thresholds (Optional)

In the PerfStat application you can configure WARN and CRIT thresholds. The thresholds are the levels at which an event is triggered. Below is how events are triggered:

WARN - A WARN event is generated when the value for a metric is at or above the WARN threshold and below the CRIT threshold

CRIT - A CRIT event is generated when the value for a metric is at or above the CRIT threshold

OK - A OK event is generated when a metric has a status of WARN or CRIT and the value falls below that of the WARN level

NOSTATUS - A NOSTATUS event is generated when the PerfStat application has not recieved data for that metric and the Status Interval (set in perf-conf) expires

1) Select "App Configs->Host Config" within the PerfStat application

2) Under the "Manage Hosts" section select "Config" to configure thresholds for a host

3) Under the "Configure Services" section select the service you would like to configure thresholds for

4) Under the service section configure the warn threshold, crit threshold, and/or threshold unit for a metric

5) Under the service section select "Update" to save your changes

Disable Service Events (Optional)

In the PerfStat application you can diable events at the service level by configuring the /perfhome/etc/events-disable file. This allows you to get more granular with your event logging policy. Disabled events can not be alerted on.

1) Using an editor such as vi open the /perfhome/etc/events-disable file.

2) Disable event(s) using the template within the file.

The events-disable file uses the following format:

hostname:<service1>:<service2>

A host followed by a comma separated list of services. You can also use the wildcard (*) to indicate all services such as follows:

hostname:*

Disable Metric Events (Optional)

In the PerfStat application you can disable events at the metric level. This allows for the most granular event policy.

1) Select "App Configs->Host Config" within the PerfStat application

2) Under the "Manage Hosts" section select "Config" to configure thresholds for a host

3) Under the "Configure Services" section select the service you would like to configure thresholds for

4) Under the service section configure the warn and/or crit thresholds for a metric

5) Under the service section select "Update" to save your changes
